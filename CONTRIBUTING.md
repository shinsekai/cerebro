# Contributing to Cerebro

We welcome contributions to Cerebro! Whether you're an AI agent, a developer, or an infrastructure engineer, helping us build the fastest multi-agent orchestration platform is deeply appreciated.

## Development Workflow

Cerebro utilizes **Turborepo** and **Bun** to manage its extremely lightweight monorepo architecture. 

### 1. Prerequisites
- [Bun v1.x](https://bun.sh)
- [Docker](https://docker.com)
- Google Cloud Vertex AI credentials for Anthropic Claude models (`ANTHROPIC_VERTEX_PROJECT_ID`).

### 2. Setup Local Environment
Clone the repository and install all nested dependencies instantly at the workspace root:
```bash
bun install
```

Boot up the required persistence layer (PostgreSQL + pgvector):
```bash
docker compose up -d
```

### 3. Running the Architecture locally
Since the Tier 2 agents require the core Hono Mesh engine to be active to handle their state schemas, you must run the API server alongside your CLI proxy tests:

```bash
# Terminal 1: Run the Backend Engine 
cd apps/engine
bun run dev
```

```bash
# Terminal 2: Test your CLI commands
cd apps/cli
bun run dev develop "Test feature"
```

## Pull Request Guidelines

1. **Adhere closely to `GEMINI.md`:** All contributions MUST strictly follow our framework constraints, including absolute adherence to the **KISS** (Keep It Simple, Stupid) and **DRY** (Don't Repeat Yourself) principles. 
2. **Circuit Breaker Integrity:** If you are modifying the `/mesh/loop` payload inside the Engine, you *must* ensure the localized agent loops remain securely bound by the `CircuitBreaker` utility. Code that is capable of infinite LLM recursion without tripping the retry limit will be instantly rejected.
3. **No `any` Types Allowed:** We enforce zero implicit or unverified `any` usage in production code. New API boundaries or payload states must be strictly validated with Zod schemas in `@cerebro/core`.

## Reporting Issues

Please open a GitHub issue detailing the exact CLI logs generated by the Engine Server when the agent crashed or failed to achieve consensus. Visuals from the `picocolors` output in the Mesh loop trace are extremely helpful.
